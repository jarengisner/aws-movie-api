Movie Findr API

The Movie Findr API is a REST API that provides information about movies, which are stored on a database. The API can provide details about individual movies, and the actors in those films.

GET REQUESTS

All GET requests send to the database pull information about different movies or users.

All GET requests accept a string parameter.

/movies

The request to /movies does not accept any parameters and will return JSON data of all movies in the database.

/movies/:title

The request to /movies/:title accepts a string that is the title of the movie, and will return JSON data containing information about that specific movie.

/movies/directors/:director

Requests to /movies/:director accept a string that is a directors full name. This will return all movies made by that director.

/movies/genres/:genreName

Requests made to /movies/:genreName accept a string that is the name of a genre. This will return all movies with that genre name.

/movies/actors/:Actors

Requests made to /movies/actors/:Actors accept a string that is the ObjectId of an actor This will return all movies with that actor in them.

/actors

The request to /actors does not accept any parameters and will return JSON data of all actors in the database.

/users

Requests made to /users will return a list of all users registered to the application.

/users/:username/favorites

Requests made to /users/:username/favorites will accept a string in the form of a users username, and will return that users list of favorites.

POST REQUESTS

All POST Requests made to the database are used to create information in the database.

/users

Post Requests made to /users are to create a new user.

Requests made to this endpoint accept a JSON object as the request body

PUT REQUESTS

All PUT Requests made to the database are used to update information in the database.

/users/:username

PUT requests made to /users/:username are used to update a user.

Requests made to this endpoint accept a JSON object as the request body, with updates applied. Only keys that are set in the JSON body to their new value will be updated, and the rest will retain the original value

/users/:username/movies/:movieId

PUT requests made to /users/:username/movies/:movieId are used to update a users favorite movies

Requests made to this endpoint accept two request parameters. :username accepts a string for a users username, and :movieId accepts the ObjectId number for a single movie.

An example request would be http://localhost:8080/users/jarengisner/movies/6433341215d053c0a3b83f4e

A JSON object of the updated user with the new movie added to their favorites would be returned if successful

DELETE REQUESTS

DELETE Requests to the database delete a piece of data

/users/:username/movies/:movieId

A DELETE request to the /users/:username/movies/:movieId endpoint will remove a movie from a users favorites

Requests made to this endpoint accept two request parameters. :username accepts a string for a users username, and :movieId accepts the ObjectId number for a single movie.

The response will return a JSON object for the user with the requested movie removed, confirming it's removal

/users/:username

A DELETE request to the /users/:username endpoint will remove a movie from a users favorites

Requests made to this endpoint accept one request parameter. :username accepts a string for a users username

The response will return confirmation that the user was deleted if successful
